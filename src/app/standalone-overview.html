Technical Overview

Component Architecture

The components have been rewritten to follow Angular 19's standalone architecture with these key principles:

  • Each component has the standalone: true property in its decorator
  • Dependencies are explicitly imported in the component instead of relying on NgModule
  • Angular Material components have been replaced with Syncfusion equivalents
  • Components leverage TypeScript's strict typing with typed reactive forms
  • CUSTOM_ELEMENTS_SCHEMA is used to handle components that haven't been migrated yet

The following feature-specific components will be migrated to standalone architecture in future sprints:

  • History components - For viewing vehicle and trip history
  • Journal components - For managing driving journals

Note: Running the development server requires additional setup to install node modules and ensure proper configurations. The modified files can be built using ng build --project=minimal command, resulting in static files that can be hosted on any web server.
